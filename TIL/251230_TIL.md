### pre-commit 사용법.

1. `$ pip install pre-commit`
2. `$ pre-commit --v` 로 pre-commit 명령어 인식유무 확인
3. `$ pre-commit sample-config > .pre-commit-config.yaml`
4. `$ pre-commit install` 하여 commit 시 자동실행하기 (강제실행은 `$ pre-commit run`)
5. 작업 -> add, commit -> pre-commit 동작 및 자동수정 -> 수정사항 add, commit -> 완료

#### https://pre-commit.com/hooks.html 추가 방법

```yaml
-   repo: https://github.com/psf/black
    rev: stable
    hooks:
    - id: black
      language_version: python3.14.2
-   repo: https://github.com/PyCQA/flake8
    rev: 7.0.0
    hooks:
    - id: flake8
```

### Merge Conflict

1. `$ git branch [브랜치]`
2. `$ git switch [브랜치]`
3. `$ git merge [브랜치]` 
4. merge conflict 해결
5. add, commit

### github flow

#### github
1. 새 issue를 생성하고 할 일에 대한 세부정보 작성(설계)

#### Local
1. 작업할 새 브랜치를 생성
2. 새 브랜치 위에서 작업 실시
3. git push origin {새 브랜치}

#### github
2. Compare & Pull Request 버튼을 눌러 새 PR(Pull Request) 생성
3. 1의 이슈번호와 연결(fix,close,resolve+{이슈번호})
4. 내용 확인 후 Merge Pull request 버튼 누르기
5. Delete Branch

#### Local
4. `$ git switch main`
5. main branch 업데이트 하기(`$ git fetch origin main` , `$ git merge FETCH_HEAD`)

의 반복(프로젝트 끝날때 까지)
